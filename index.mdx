A Relynt **extension** is a complete integration — a declarative package that connects your systems to the platform. Each extension is a bundle of YAML manifests that define capabilities, workflows, knowledge entities, and UI panels. You author manifests locally, validate them with the CLI, and publish immutable versions to the extension registry.

## What an extension contains

| Component | Purpose |
|-----------|---------|
| **Capabilities** | Typed operations with input/output schemas and HTTP or endpoint bindings |
| **Workflows** | Step graphs for email intake and work item automation |
| **Knowledge** | Entity types and sync mappings for structured data |
| **Panels** | Declarative UI surfaces rendered at runtime from CEL bind values |

## How it works

1. **Author** — Write YAML manifests in a bundle directory.
2. **Validate** — Run `relynt validate` locally using the same rules as the server.
3. **Publish** — Push an immutable semver version with `relynt push`.
4. **Install** — Pin a version for your organization with `relynt install`.

<Note>
  Bundle workflows are validated and stored in the registry but are **not imported on install**. The installing organization must adopt workflow templates through the workflow builder before automation runs. See [Lifecycle](/extensions/platform/lifecycle) for details.
</Note>

## Prerequisites

- The [Relynt CLI](/cli/overview) installed on your machine
- A Relynt organization with extension publish permissions
- Basic familiarity with YAML and JSON Schema
