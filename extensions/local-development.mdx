## Install the CLI

<Note>
  Binary downloads and package manager installs (Homebrew, apt, and more) are coming soon.
</Note>

## Authenticate

Log in with your organization before pushing or installing:

```bash
relynt auth login --org <org-id>
relynt auth whoami
```

See [CLI auth](/cli/auth) for subcommands and flags.

## Validate locally

Run validation before pushing. The CLI uses the same validation engine as the server:

```bash
relynt validate ./manifests
```

Fix any errors reported with `{path, code, message}` before continuing.

## Use in CI

Validation does not require network access — use it in pre-commit hooks or CI pipelines:

```bash
relynt validate ./manifests
relynt validate ./manifests --json
```

JSON output is machine-readable for automated error reporting.
