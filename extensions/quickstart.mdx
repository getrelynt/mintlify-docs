title: "Quickstart"
description: "Scaffold, validate, and publish your first Relynt extension."
---

Get a working extension from zero to published in five steps.

## Prerequisites

- [Relynt CLI](/cli/overview) installed
- Authenticated with `relynt auth login` (see [Local development](/extensions/local-development))

<Steps>
  <Step title="Scaffold a bundle">
    Create a new extension bundle with the CLI:

    ```bash
    relynt init --name my-extension
    ```

    This creates a `manifests/` directory with a starter `extension.yaml`, a README, and empty `workflows/`, `panels/`, and `knowledge/` folders.
  </Step>
  <Step title="Edit manifests">
    Add capabilities, workflows, knowledge, and panels to your extension. At minimum, your bundle must include one `kind: Extension` manifest that sets the extension identity (`metadata.name` and `metadata.version`).
  </Step>
  <Step title="Validate locally">
    Run validation before pushing. The CLI uses the same validation engine as the server:

    ```bash
    relynt validate ./manifests
    ```

    Fix any errors reported with `{path, code, message}` before continuing.
  </Step>
  <Step title="Authenticate">
    Log in with your organization:

    ```bash
    relynt auth login --org <org-id>
    relynt auth whoami
    ```
  </Step>
  <Step title="Push and install">
    Publish an immutable version and install it for your org:

    ```bash
    relynt push ./manifests
    relynt install my-extension@0.1.0
    ```

    `push` validates locally first, then sends the bundle to `POST /v1/extensions/bundle`.
  </Step>
</Steps>
